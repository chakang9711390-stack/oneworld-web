# 一世界｜V1 前端页面开发规范与组件拆分清单 v1

## 1. 目标

本文件用于指导前端将当前高保真原型拆为真实可维护的页面结构与组件系统，避免直接复制 HTML 造成后续难维护、难扩展的问题。

---

## 2. 页面开发规范

### 2.1 页面职责清晰
每个页面只负责自己的主目标，不要让一个页面承担过多业务逻辑。

### 2.2 组件优先复用
以下内容必须组件化，不允许重复硬编码：
- 导航按钮
- 卡片样式
- 搜索框
- 面包屑
- 按钮
- 弹窗
- 指标卡片

### 2.3 样式变量统一管理
颜色、阴影、圆角、间距统一收敛到主题变量，不允许在页面中散落大量魔法值。

### 2.4 页面数据与展示分离
页面负责拼装，数据读取逻辑尽量收敛在 feature 层或 hooks 层。

## 3. 推荐页面拆分

### 3.1 首页 `/`
建议拆分组件：
- `TopNav`
- `HeroSection`
- `SearchBar`
- `EntryCard`
- `EntryCardGrid`

### 3.2 OPC 行业库 `/opc`
建议拆分组件：
- `SectionHeader`
- `SearchBar`
- `IndustryCard`
- `IndustryCardGrid`

### 3.3 OPC 项目卡片页 `/opc/[industrySlug]`
建议拆分组件：
- `Breadcrumb`
- `SectionHeader`
- `OpcProjectCard`
- `CreateOpcCard`
- `ProjectCardGrid`

### 3.4 工作流行业库 `/workflow`
建议拆分组件：
- `SectionHeader`
- `SearchBar`
- `IndustryCard`
- `IndustryCardGrid`

### 3.5 工作流职位页 `/workflow/[industrySlug]`
建议拆分组件：
- `Breadcrumb`
- `SectionHeader`
- `WorkflowRoleCard`
- `RoleCardGrid`

### 3.6 工作流场景页 `/workflow/[industrySlug]/[roleSlug]`
建议拆分组件：
- `Breadcrumb`
- `SectionHeader`
- `WorkflowSceneCard`
- `CreateWorkflowCard`
- `SceneCardGrid`

### 3.7 工作台 `/workspace`
建议拆分组件：
- `SectionHeader`
- `PrimaryActionButton`
- `MetricCard`
- `MetricGrid`

### 3.8 登录注册弹窗
建议拆分组件：
- `AuthModal`
- `AuthTabs`
- `LoginForm`
- `SignupForm`
- `OAuthButtons`

## 5. Feature 目录建议

建议按功能域拆分：

- `features/home`
- `features/opc`
- `features/workflow`
- `features/workspace`
- `features/auth`

每个 feature 内可包含：
- `components/`
- `hooks/`
- `services/`
- `types/`

---

## 6. 状态规范

建议统一考虑以下状态：
- loading
- empty
- disabled
- active
- logged-in
- guest

尤其当前原型里大量卡片属于“可浏览、暂不可点击”的状态，前端实现时必须显式设计，不要模糊处理。

---

## 7. 代码实现原则

1. 不直接照搬原型 HTML
2. 先抽布局，再抽组件，再接数据
3. 样式主题化
4. 卡片与列表数据结构化
5. 路由参数化，避免写死

---

## 8. 最终结论

前端 V1 的重点不是“还原一张图”，而是：

> 把一世界当前高保真原型，拆成未来还能持续迭代的平台型前端工程。
